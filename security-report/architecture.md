# UWAS Architecture Map

Generated by security-check reconnaissance phase.

## 1. Technology Stack Detection

### Languages
- **Go** (~65% of codebase) ‚Äî Primary backend language, all server logic
- **TypeScript** (~30% of codebase) ‚Äî React 19 dashboard SPA
- **YAML** (~5% of codebase) ‚Äî Configuration files, domain configs

### Frameworks & Libraries
- **Go**: Custom HTTP server using `net/http` (stdlib), `quic-go` for HTTP/3
- **TypeScript**: React 19, React Router DOM 7, Tailwind CSS 4, Vite 8, Recharts, XYFlow
- **Build**: Custom CLI framework (not Cobra), Go 1.26+ modules

### Build Tools & Package Managers
- `go.mod` / `go.sum` ‚Äî Go modules
- `package.json` / `npm` ‚Äî Node.js dashboard dependencies
- `Makefile` ‚Äî Build orchestration

### Databases
- MySQL/MariaDB (managed via Docker containers or system packages)
- No ORM; raw SQL via `database/sql` or exec commands
- Redis (optional L3 cache layer)

## 2. Application Type Classification

**Type**: Web Application + Reverse Proxy Server + Admin Control Panel

Indicators:
- HTTP/HTTPS/HTTP3 server with virtual host routing
- React 19 SPA dashboard embedded via `go:embed`
- 205+ REST API endpoints for server management
- 19 CLI commands for system operations
- Monolithic architecture (single binary ~15MB)

## 3. Entry Points Mapping

### HTTP Routes (205+ Admin API endpoints)
File: `internal/admin/api.go:registerRoutes()`

Key route categories:
- **Public**: `/api/v1/health`, `/api/v1/auth/login`, `/api/v1/auth/ticket`
- **Auth Required**: All other `/api/v1/*` endpoints
- **WebSocket**: `/api/v1/terminal` (WebSocket-to-PTY bridge, requires PIN)
- **SSE**: `/api/v1/sse/stats`, `/api/v1/sse/logs`
- **Dashboard SPA**: `/_uwas/dashboard/*` (embedded static files)
- **Metrics**: `/api/v1/metrics` (Prometheus-compatible)

### CLI Commands (19 commands)
File: `cmd/uwas/main.go` registers commands in `internal/cli/`

Commands: serve, stop, restart, status, version, cert, config, php, user, migrate, doctor, backup, restore, update, install, help

### WebSocket Endpoints
- `/api/v1/terminal` ‚Äî Browser-based shell (requires `requirePin` check)

### File Watchers / Event Handlers
- `.htaccess` file watcher (modTime tracking, auto-invalidation)
- Config file watchers for domain reloads
- PHP process crash restart watchers

## 4. Data Flow Map

### Sources
- HTTP request body, query params, headers, cookies, URL path segments
- File uploads (`/api/v1/files/{domain}/upload`)
- CLI arguments and flags
- Environment variables (`.env` files at `~/.uwas/.env`, `/etc/uwas/.env`)
- Config files (`uwas.yaml`, `domains.d/*.yaml`)

### Processing
- Request ID generation (`internal/middleware/chain.go`)
- Security headers injection
- Rate limiting (global + per-domain)
- IP ACL / GeoIP filtering
- WAF body scan (first 64KB)
- `.htaccess` parsing and rewrite engine
- Cache lookup (L1 memory ‚Üí L2 disk ‚Üí L3 Redis)

### Sinks
- Database queries (MySQL via `database/sql` or command execution)
- File system operations (static handler, file manager, backups)
- Shell command execution (PHP manager, app manager, deploy, install)
- HTTP responses (HTML, JSON, SSE)
- Outbound HTTP requests (proxy handler, webhook delivery, DNS APIs)
- Logging (structured slog wrapper)

## 5. Trust Boundaries

### Authentication
- **Session-based**: Session tokens stored in `auth.Manager` in-memory map
- **API Key**: SHA256 hash comparison with `crypto/subtle.ConstantTimeCompare`
- **Basic Auth**: Per-domain basic auth via `internal/middleware/basicauth.go`
- **TOTP 2FA**: Time-based OTP via `X-TOTP-Code` header
- **JWT**: In-memory JWT secret (32 bytes from `crypto/rand`), not persisted

### Authorization
- RBAC with 3 roles: `admin`, `reseller`, `user`
- Permission checks: `HasPermission(role, perm)` + `CanManageDomain(user, domain)`
- Per-domain scoping for resellers

### Rate Limiting
- Global middleware rate limiter
- Per-domain rate limiters (configurable)
- Per-username login failure rate limiting
- Auth failure rate limiting in admin server

### Input Validation
- Domain hostname regex validation (rejects injection/traversal chars)
- Path traversal guard (`internal/pathsafe/`)
- Custom validation functions in admin handlers
- No formal schema validation library (Zod/Joi equivalent)

### CSRF Protection
- Not explicitly implemented for API endpoints (relies on session/API key auth)
- SameSite cookie not documented

### CORS Configuration
- Configurable per-domain CORS middleware (`internal/middleware/cors.go`)
- Admin API does not appear to have explicit CORS protection

## 7. Authentication Architecture

**Pattern**: Multi-model hybrid

| Method | Implementation |
|--------|---------------|
| Session | In-memory map with token ‚Üí Session, 24h expiry |
| API Key | SHA256 hash, prefix displayed, full key shown once at generation |
| Basic Auth | Per-domain via middleware, bcrypt password comparison |
| TOTP 2FA | Standard TOTP, recovery codes supported |
| Global API Key | Backward compatibility single key in config |

Password hashing: `bcrypt` with `bcrypt.DefaultCost`
Session storage: In-memory only (not persistent across restarts)
Ticket system: Short-lived single-use tickets for SSE/WebSocket (avoids token in URL)

## 9. Detected Security Controls

- **WAF**: URL + request body inspection (first 64KB), SQL/XSS/shell/RCE detection
- **Bot Guard**: Blocks 25+ malicious scanners
- **Rate Limiting**: Global and per-domain configurable
- **Security Headers**: Injected by middleware (HSTS, X-Frame-Options, CSP, etc.)
- **Path Traversal Guard**: Symlink-resolving containment (`internal/pathsafe/`)
- **PHP Sandbox**: `disable_functions`, `open_basedir`, `allow_url_include=Off`
- **TLS Manager**: ACME auto-issuance, cert expiry alerts
- **IP ACL**: Per-domain allow/deny lists
- **Bandwidth Throttling**: Per-domain limits
- **Audit Logging**: Security event ring buffer
- **Credential Generation**: `crypto/rand.Read` with panic on failure
